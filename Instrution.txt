Challenge Transactions Backup and Recovery

Repository contents
- `transactions.sql`  - Examples of transactions without a procedure.
- `procedure_transaction.sql` - Procedure with transaction, savepoint and error handling.
- `sample_schema.sql` - Optional script to create a minimal test schema (products, customers, orders, order_items).
- `backups/` - Suggested folder for backup files (e.g., `ecommerce_full_backup_2026-04-09.sql.gz`).
- `README.md` - Project instructions.

How to use the SQL scripts
1. Connect to MySQL as a user with appropriate privileges.  
2. (Optional) Create a test database and run `sample_schema.sql` to populate tables.  
3. Run `transactions.sql` to test manual transactions.  
4. Run `procedure_transaction.sql` to create the stored procedure and test it with a call such as:  
   ```sql
   CALL sp_create_order_with_items(1, 101, 2, 49.90, 102, 1, 29.90);

Backup with mysqldump
- Full backup including routines events and triggers:
  `mysqldump -u root -p --routines --events --triggers --databases ecommerce > ecommerce_full_backup.sql`
- Consistent backup for InnoDB:
  `mysqldump -u root -p --single-transaction --routines --events --triggers --databases ecommerce > ecommerce_consistent.sql`
- Compress backup:
  `mysqldump ... | gzip > ecommerce_full_backup.sql.gz`

Restore
- Simple restore:
  `mysql -u root -p < ecommerce_full_backup.sql`
- Restore from compressed file:
  `gunzip < ecommerce_full_backup.sql.gz | mysql -u root -p`

Important notes

- Test restores in a staging or homologation environment before applying to production.
- Do not store large backups directly in a Git repository. Use object storage or Git LFS for large files.
- If you include backups in the repo for the exercise, document file size and date in the README.
- Use a user with the required privileges to restore routines events and triggers: CREATE ROUTINE, ALTER ROUTINE, EVENT, and appropriate table privileges.
- For high activity databases, prefer --single-transaction for consistent dumps of InnoDB tables and avoid --lock-tables.
